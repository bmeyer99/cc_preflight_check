## Task Details
### Requirements:
- Modify the script to accept new command-line arguments (e.g., `--condition-values "IsProd=true,IsDev=false"` or a similar mechanism).
- These arguments will allow users to provide input values for parameters or pseudo-parameters used within the `Conditions` block of a CloudFormation template.
- Store these user-provided condition values in a way that they can be accessed during condition evaluation.

### Acceptance Criteria (AC):
- The script successfully parses and stores condition values provided via new CLI arguments.
- The mechanism for providing condition values is clearly documented for the user (e.g., in help text or [`README.md`](README.md)).
- The script handles various input formats for condition values gracefully (e.g., multiple key-value pairs).
- If no condition values are provided by the user, the script proceeds without error, potentially treating undefined conditions in a default manner (to be defined in P2-T2.3).
